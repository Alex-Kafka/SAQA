# Хореография и Оркестрация

**Оркестрация** - централизованное управление бизнес-процессом, когда есть единственный координатор (оркестратор), который вызывает необходимые сервисы.

**Хореография** - децентрализованное управление, когда каждый сервис самостоятельно реагирует на события и выполняет свою часть работы.

### Сравнение
| Параметр | Оркестрация | Хореография |
|----------|-------------|-------------|
| **Управление** | Централизованное | Децентрализованное |
| **Сложность** | Проще для отслеживания процесса | Сложнее для отслеживания |
| **Связность** | Сервисы зависят от оркестратора | Сервисы не знают друг о друге |
| **Гибкость** | Изменения в процессе требуют изменения оркестратора | Легче добавлять новые сервисы |

### Примеры
- **Оркестрация:** AWS Step Functions, Apache Airflow
- **Хореография:** Event-driven архитектура с Kafka
