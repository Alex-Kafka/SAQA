
# Поисковые пути в БД

**Поисковый путь (Search Path)** - это список схем в базе данных, в котором СУБД ищет объекты (таблицы, функции и т.д.), если схема не указана явно.

### Пример (PostgreSQL)

-- Показать текущий поисковый путь
SHOW search_path;

-- Установить поисковый путь
SET search_path TO schema1, schema2, public;

### Зачем нужен?

- Упрощает запросы, позволяя не указывать схему для объектов.
    
- Позволяет иметь несколько объектов с одинаковыми именами в разных схемах, но обращаться к тому, который раньше в пути.
    

### Важно

- Если объект не найден в указанных схемах, будет ошибка.
    
- Первая схема в пути, где найден объект, и будет использована.
