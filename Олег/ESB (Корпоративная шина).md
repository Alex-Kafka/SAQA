# ESB (Корпоративная шина)

**Enterprise Service Bus (ESB)** — это архитектурный паттерн, который представляет собой централизованный механизм для интеграции различных приложений и сервисов в рамках предприятия. ESB выступает в роли "шины", к которой подключаются все системы.

### Основные функции
- **Маршрутизация сообщений:** Определение, куда должно быть доставлено сообщение.
- **Трансформация данных:** Преобразование форматов данных между системами (например, XML в JSON).
- **Протокольная адаптация:** Обеспечение взаимодействия между системами, использующими разные протоколы (HTTP, JMS, FTP и т.д.).
- **Обеспечение безопасности:** Аутентификация и авторизация запросов.
- **Мониторинг и управление:** Контроль за потоками сообщений и обработка ошибок.

### Аналогия
ESB — это как центральный автовокзал, который:
- Принимает автобусы (сообщения) из разных городов (систем).
- Перенаправляет их по нужным маршрутам.
- Меняет пассажиров (данные) с одного автобуса на другой, если нужно.

### Что изменится при добавлении новых полей?
Если к веб-сервису, подключенному к ESB, добавляются два новых обязательных поля:
1.  **Изменяется XSD-схема** этого сервиса.
2.  **ESB необходимо перенастроить:**
    - Трансформации данных для этого сервиса должны быть обновлены, чтобы включать новые поля.
    - Может потребоваться обновить маршрутизацию, если новые поля влияют на логику маршрутизации.
3.  **Все клиенты**, которые через ESB обращаются к этому сервису, должны будут передавать эти поля (или ESB должен будет их подставлять, если это возможно).
