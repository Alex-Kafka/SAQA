# REST

**REST (Representational State Transfer)** — это архитектурный стиль для построения распределенных систем, основанный на принципах HTTP.

### Ключевые принципы
1.  **Stateless:** Сервер не хранит состояние клиента между запросами.
2.  **Единообразие интерфейса:** Стандартные HTTP-методы (GET, POST, PUT, DELETE) и коды ответов.
3.  **Ресурсо-ориентированность:** Все сущности представлены как ресурсы с URI.
4.  **Кэшируемость:** Ответы должны явно указывать, можно ли их кэшировать.

### Пример REST API
- `GET /api/v1/users` - получить список пользователей
- `POST /api/v1/users` - создать нового пользователя
- `GET /api/v1/users/123` - получить пользователя с ID=123
- `PUT /api/v1/users/123` - обновить пользователя с ID=123
- `DELETE /api/v1/users/123` - удалить пользователя с ID=123

### Версионирование API
- **URI версионирование:** `https://api.example.com/v1/users`
- **Заголовок версионирование:** `Accept: application/vnd.example.v1+json`
- **Параметр запроса:** `https://api.example.com/users?version=1`
