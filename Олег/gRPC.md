# gRPC

**gRPC** — это современный фреймворк для удаленного вызова процедур (RPC), разработанный Google. Использует HTTP/2 и Protocol Buffers (protobuf) как язык описания интерфейса.

### Как работает?
1.  **Определение интерфейса** в .proto-файле с помощью Protocol Buffers.
2.  **Генерация кода** для клиента и сервера на разных языках.
3.  **Вызов методов** как локальных функций.

### Преимущества перед REST
- **Высокая производительность:** Бинарный формат, HTTP/2, мультиплексирование.
- **Строгая типизация:** Интерфейс четко определяется в .proto-файле.
- **Потоковая передача:** Поддержка двунаправленных потоков.
- **Встроенная генерация кода.**

### Недостатки
- **Сложнее для отладки** (бинарный формат).
- **Менее "веб-дружелюбный"** чем REST/JSON.

### Когда использовать?
- Внутренняя коммуникация между микросервисами.
- Системы, требующие высокой производительности.
- Приложения с потоковой передачей данных.
