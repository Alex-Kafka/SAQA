# Требования

**Требование** — это условие или возможность, необходимая пользователю для решения проблемы или достижения цели. Требования являются основным продуктом работы бизнес-аналитика.

## Классификация по BABOK (Business Analysis Body of Knowledge)
1.  **Бизнес-требования (Business Requirements)** - Высокоуровневые цели организации или заказчика. Описывают, *почему* инициирован проект. Фиксируются в [[V&S (Vision and Scope)|Vision and Scope]] или [[BRD]].
    *   *Пример: "Увеличить долю рынка на 15% за счет запуска онлайн-заказов".*

2.  **Пользовательские требования (Stakeholder Requirements)** - Ожидания конкретных пользователей или ролей от системы. Описывают, *что* пользователь должен быть способен делать.
    *   *Пример: "Менеджер по продажам должен иметь возможность сформировать отчет по клиентам за выбранный период".*

3.  **Функциональные требования (Functional Requirements)** - Конкретное поведение и функции системы, которые реализуют пользовательские требования. Описывают, *как* система должна работать.
    *   *Пример: "Система должна предоставлять REST API метод `/api/v1/reports/customers` для получения данных о клиентах".*

4.  **Нефункциональные требования (Non-Functional Requirements)** - Описывают атрибуты качества системы и ограничения среды. Отвечают на вопрос *насколько хорошо* система должна работать.
    *   *Примеры: Производительность, безопасность, надежность, удобство использования.*

## Классификация по Карлу Вигерсу (Karl Wiegers)
-   **Бизнес-правила (Business Rules)** - Политики, стандарты, алгоритмы, которые определяют поведение системы.
-   **Функциональные требования** - Аналогично BABOK.
-   **Атрибуты качества (Quality Attributes)** - Аналогично нефункциональным требованиям.
-   **Ограничения (Constraints)** - Факторы, ограничивающие решения (например, "должно работать в браузере IE11").
-   **Внешние интерфейсы (External Interfaces)** - Требования к взаимодействию с другими системами.

### Связи между требованиями
БА настраивает связи с помощью специализированных инструментов (Jira, Confluence, специализированные RM-системы) или простых таблиц:
- **Родитель-потомок** (Эпик -> User Story -> Задача)
- **Зависимости** (Требование А не может быть реализовано без Требования Б)
- **Связь "реализует"** (Функциональное требование реализует Пользовательское требование)
- **Связь "влияет на"** (Изменение в одном требовании влияет на другие)
