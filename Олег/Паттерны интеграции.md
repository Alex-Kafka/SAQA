# Паттерны интеграции

**Паттерны интеграции** — это стандартные, проверенные способы организации взаимодействия между различными программными системами.

## Основные способы интеграции

### 1. Файловый обмен
Системы обмениваются данными через файлы (CSV, XML, JSON) в общей файловой системе.
- *Плюсы:* Простота реализации.
- *Минусы:* Задержки, проблемы с блокировками, слабая надежность.

### 2. Общая база данных
Несколько систем используют одну базу данных.
- *Плюсы:* Высокая производительность, гарантия консистентности.
- *Минусы:* Тесная связность, риск блокировок, сложность миграции.

### 3. Удаленный вызов процедур (RPC)
Системы вызывают методы друг друга как локальные.
- **Синхронные:** REST, SOAP, gRPC
- **Асинхронные:** Брокеры сообщений

### 4. Обмен сообщениями
Асинхронная коммуникация через брокер сообщений.
- *Плюсы:* Слабая связность, высокая надежность, буферизация.
- *Минусы:* Сложность, дополнительные infrastructural components.

## Паттерны для микросервисов

### Хореография vs Оркестрация
- **Хореография:** Каждый сервис самостоятельно реагирует на события (децентрализовано). *Пример:* Система уведомлений получает событие "Заказ создан" и отправляет email.
- **Оркестрация:** Есть центральный координатор (оркестратор), который управляет бизнес-процессом и вызывает нужные сервисы. *Пример:* Сервис "Оформление заказа" последовательно вызывает сервисы "Проверка наличия", "Списание средств", "Создание доставки".
